- 尊重事实比尊重我更加重要
- 项目当中所有的文件使用LF作为行尾符,不要使用CRLF作为行尾符
- 执行代码生成任务的时候应当注意生成代码的简洁高效优雅
- 任务完成后需要自动检查IDE的报错与警告，如果存在必须消除所有报错与警告
- 在排查错误的过程中如果我出现了拼写错误这种不易识别的错误你必须明确告知我,但是你不必主动为我更改拼写错误
- 如果打算输出表格,不要输出Markdown格式的表格因为不能被正确渲染,请改用其他形式
- 当一个函数中存在多个功能相对独立的代码块的时候,应该使用 region folding comments 进行分块组织代码,使其更加易读和维护
- 如果文件当中已经存在 一些 region folding comments，生成新代码的时候应该考虑是否放置在已经存在的 region 内部
